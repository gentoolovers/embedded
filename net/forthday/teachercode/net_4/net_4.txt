【1】poll 实现并发服务器
     int poll(struct pollfd *fds, nfds_t nfds, int timeout);
      功能：与select 相似。阻塞等待被监听文件描述符准备就绪。如果准备就绪就立刻返回（不考虑timeout）。
      参数：fds   要监听的文件描述符的数组
            nfds  监听的文件描述符的个数
            timeout  超时时间值
      返回值：成功 已经准备就绪的文件描述符的个数
                   如果timeout 时间到达时，还没有资源准备就绪，那么返回值为0
              失败 -1

       struct pollfd {
               int   fd;         /* file descriptor */  要监听的文件描述符
               short events;     /* requested events */ 感兴趣的事件
               short revents;    /* returned events */  实际发生的事件
       };

【3】网络超时检测
     超时检测的必要性
      1-- 避免进程在没有数据时无限制地阻塞
      2-- 当设定的时间到时，进程从原操作返回继续运行

     1-- 使用setsockopt 设置套接字属性，具有全局性，一次设置终身有效。 实现超时检测

     struct timeval
     {
       __time_t tv_sec;        /* Seconds.  */
       __suseconds_t tv_usec;  /* Microseconds.  */
      };                   

     设置socket的属性 SO_RCVTIMEO

     参考代码如下
     struct timeval  tv;

     tv.tv_sec = 5;   //  设置5秒时间
     tv.tv_usec = 0;

     setsockopt(sockfd, SOL_SOCKET, SO_RCVTIMEO,  &tv,  
                       sizeof(tv));     //  设置接收超时
      recv() / recvfrom()          //   从socket读取数据

     2-- 使用select 实现超时检测。

      int select(int nfds, fd_set *readfds, fd_set *writefds,
                  fd_set *exceptfds, struct timeval *timeout);
       功能：允许进程监听多个文件描述符集合，它一直阻塞等待，
             直到集合中有一个或者多个文件描述符准备就绪，那么函数会立刻返回。
             如果timeout 不为NULL，表示阻塞等待的时间，如果在规定的时间内，没有资源
             准备就绪，那么函数一直阻塞直到时间到达，返回值为0，如果有资源准备就绪立刻返回。

       参数：nfds  最大文件描述加1
             readfds  读文件描述资源
             writefds 写资源
             exceptfds 异常
             timeout   NULL 一直阻塞
       返回值：成功 已经准备就绪的文件描述符个数                        
               出错 -1


        struct timeval  tv;
         tv.tv_sec = 5;
         tv.tv_usec = 0;
  
        if((ret = select(maxfd+1, &readfds, NULL, NULL, &tv)) < 0)
         {
              err_log("fail to select");
          }                                                                                                         
  
        注意：每次select 执行完毕之后，tv的值会被清零。

【4】广播
     前面介绍的数据包发送方式只有一个接受方，称为单播

     如果同时发给局域网中的所有主机，称为广播

     只有用户数据报(使用UDP协议)套接字才能广播

     广播地址
     以192.168.1.0 (255.255.255.0) 网段为例，最大的主机地址192.168.1.255代表该网段的广播地址
     发到该地址的数据包被所有的主机接收
     255.255.255.255在所有网段中都代表广播地址


    广播发送者：
    1-- 创建用户数据报套接字

    2-- 设置发送广播权限
        缺省创建的套接字不允许广播数据包，需要设置属性
        setsockopt可以设置套接字属性

    3-- 填充广播网络信息结构体
        接收方地址指定为广播地址
        指定端口信息

    4-- 发送数据包

    广播接收者：
    1-- 创建用户数据报套接字
     
    2-- 绑定
        绑定广播IP地址和端口
        绑定的端口必须和发送方指定的端口相同

    3-- 等待接收数据

    对于广播来说，发送者需要权限。

【5】组播（又叫多播）
     单播方式只能发给一个接收方。

     广播方式发给所有的主机。过多的广播会大量占用网络带宽，造成广播风暴，影响正常的通信。

     组播(又称为多播)是一种折中的方式。只有加入某个多播组的主机才能收到数据。

     多播方式既可以发给多个主机，又能避免象广播那样带来过多的负载(每台主机要到传输层才能判断广播包是否要处理)

     A类地址
     第1字节为网络地址，其他3个字节为主机地址。第1字节的最高位固定为0
     1.0.0.1 C 126.255.255.255

     0.0.0.0  代表任意本机地址（0 开头被保留了）
     127.0.0.0 -- 127.255.255.255 （被保留了）  ， 127.0.0.1  用来本地回环地址。

     B类地址
     第1字节和第2字节是网络地址，其他2个字节是主机地址。第1字节的前两位固定为10
     128.0.0.1 C 191.255.255.255

     10 00 0000  0000 0000 -- 10 11 1111 1111 1111

     C类地址
     前3个字节是网络地址，最后1个字节是主机地址。第1字节的前3位固定为110
     192.0.0.1 C 223.255.255.255

     D类地址（组播地址）
     不分网络地址和主机地址，第1字节的前4位固定为1110
     224.0.0.1 C 239.255.255.255


    A类中的私有地址和保留地址：
     10.0.0.0 C 10.255.255.255 是私有地址，仅用于局域网
     127.0.0.0 C 127.255.255.255 是保留地址，用于循环测试用
     0.0.0.0 C 0.255.255.255 也是保留地址，用所表示所有的IP地址
    D类地址中224.0.0.0 C 224.0.0.255是被保留的地址

    
 组播发送：
    1-- 创建用户数据报套接字

    2-- 填充组播网络信息结构体
        接收方地址指定为组播地址
        指定端口信息

    3-- 发送数据包

 组播接收：
    1-- 创建用户数据报套接字

    2-- 加入多播组
        setsockopt();   // IPPROTO_IP

    3-- 绑定组播的地址
       绑定组播IP地址和端口
       绑定的端口必须和发送方指定的端口相同

    4-- 等待接收数据
